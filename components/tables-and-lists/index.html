Tables and Lists

Tables

Tables have no base styling in Sandstone (apart from the browser defaults). Use the .table class to display tabular data with a bit more polish.

Table caption
Column Column Column
Row Cell Cell Cell
Row Cell Cell Cell

HTML


<table class="table">
<caption>Table caption</caption>
<thead>
<tr>
  <td></td>
  <th scope="col">Column</th>
  <th scope="col">Column</th>
  <th scope="col">Column</th>
</tr>
</thead>
<tbody>
<tr>
  <th scope="row">Row</th>
  <td>Cell</td>
  <td>Cell</td>
  <td>Cell</td>
</tr>
<tr>
  <th scope="row">Row</th>
  <td>Cell</td>
  <td>Cell</td>
  <td>Cell</td>
</tr>
</tbody>
</table>

Unstyled lists

It’s often handy to use an ordered or unordered list in for semantic markup while avoiding any default styling, either from the browser or from Sandstone. Use the .unstyled class to override the default styling. These list items have no markers or indentation.

  • list item
  • list item

HTML


<ul class="unstyled">
<li>list item</li>
<li>list item</li>
</ul>

Definition lists

Terms in standard definition lists recieve the same general styling as top-level headings (h1). This doesn’t make a lot of sense and is almost always undesirable, hence why it seems alternate styles for definition lists were created over time. We should fix this in Sandstone, but I’m documenting it here for now.

Term
Description

HTML


<dl>
<dt>Term</dt>
<dd>Description</dd>
</dl>

Definition lists, simple

Simpler definition lists, with more reasonably sized terms. This should probably be the base style for this element.

Term
Description

HTML


<dl class="simple">
<dt>Term</dt>
<dd>Description</dd>
</dl>

Frequently Asked Questions

Lists of questions and their answers are marked up as a definition list.

Is this a question?
Yes. Yes it is.

HTML


<dl class="faq">
<dt>Is this a question?</dt>
<dd>Yes. Yes it is.</dd>
</dl>
